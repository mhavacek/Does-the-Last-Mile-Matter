# Does the Last Mile Matter?

This repository contains a reproducible Python analysis of first-mile and last-mile access/egress burden among public transit trips in the 2017 U.S. National Household Travel Survey (NHTS). The workflow filters public-use trip records to transit trips, constructs access and egress burden measures from NHTS access/egress variables, produces descriptive tables and figures, and fits exploratory classification models. The analysis is observational and should be interpreted as descriptive, associational, and predictive rather than causal.

## Data Source

The analysis uses the 2017 National Household Travel Survey public-use files. Download the data from the NHTS website:

<https://nhts.ornl.gov>

Download the 2017 NHTS Survey Data in CSV format, then place the extracted files in `data/raw/2017/`:

```text
data/raw/2017/trippub.csv
data/raw/2017/perpub.csv
data/raw/2017/hhpub.csv
data/raw/2017/vehpub.csv
```

The raw NHTS files are public-use data but are not redistributed in this repository. See [data/README.md](data/README.md) for details.

Recommended citation:

Federal Highway Administration. (2017). 2017 National Household Travel Survey, U.S. Department of Transportation, Washington, DC. Available online: <https://nhts.ornl.gov>.

## Software Requirements

The analysis was run with Python 3.11.15. Package versions observed in the local analysis environment are listed in `requirements.txt`.

To create a fresh environment:

```bash
python3.11 -m venv .venv
.venv/bin/python -m pip install --upgrade pip
.venv/bin/python -m pip install -r requirements.txt
```

If your shell uses a different Python command, replace `python3.11` with the path to a Python 3.11 interpreter.

## Reproduce the Analysis

1. Clone the repository.
2. Download the 2017 NHTS Survey Data CSV files from <https://nhts.ornl.gov>.
3. Extract and place the files under `data/raw/2017/` as listed above.
4. Install Python requirements.
5. Run the full workflow:

```bash
.venv/bin/python run_all.py
```

Each step can also be run independently:

```bash
.venv/bin/python src/01_load_and_inspect.py
.venv/bin/python src/02_prepare_analysis_dataset.py
.venv/bin/python src/03_descriptive_analysis.py
.venv/bin/python src/04_modeling.py
.venv/bin/python src/05_generate_tables_and_figures.py
```

## Outputs

The workflow generates:

- `data/processed/nhts_transit_last_mile_analysis.csv`: cleaned trip-level analysis dataset.
- `outputs/logs/variable_availability.txt`: detected variables and availability notes.
- `outputs/tables/*.csv`: sample flow, descriptive tables, model results, metadata, and missing-data notes.
- `outputs/figures/*.png`: generated figures.
- `figures/*.png`: article-ready figures.
